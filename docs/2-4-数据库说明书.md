# 数据库说明书

## 1. 引言

### 1.1 编写目的

本文档描述"Golang AI 原生全栈应用快速开发脚手架"所生成项目的数据库设计。由于脚手架本身是一个命令行工具，不直接持有业务数据库，因此本文档重点说明：

1. 脚手架生成项目（以 Todo Demo 为例）的数据库设计；
2. 脚手架自身可能使用的元数据存储设计；
3. 数据库选型、连接配置、迁移策略和索引设计。

### 1.2 数据库选型

生成项目默认使用 **PostgreSQL 15+**，主要基于以下考虑：

1. **类型安全**：PostgreSQL 支持丰富的数据类型，与 Ent 的类型系统配合良好。
2. **开源免费**：适合课程实训和中小型项目。
3. **生态成熟**：与 Docker、云服务商兼容性良好，便于部署。
4. **扩展性强**：支持 JSONB、全文检索等高级特性，便于后续功能扩展。

同时，系统也支持 MySQL 8.0+ 和 SQLite 3 作为可选项，以满足不同场景需求。

### 1.3 术语定义

| 术语 | 说明 |
|:---|:---|
| Schema | 数据库模式，描述表、字段、索引、关系等结构 |
| Ent | Go 语言的实体框架，通过代码生成实现类型安全的数据库操作 |
| Migration | 数据库迁移，用于管理数据库 Schema 的变更 |
| DSN | Data Source Name，数据库连接字符串 |
| PK | Primary Key，主键 |
| FK | Foreign Key，外键 |
| Index | 索引，用于加速数据查询 |

### 2.2 数据库 ER 图说明

Todo Demo 应用包含两个核心实体：

1. **用户（User）**：代表系统的注册用户，包含用户名、邮箱、密码哈希等字段。
2. **待办事项（Todo）**：代表用户创建的待办任务，包含标题、描述、完成状态、截止日期等字段，并通过 `user_id` 与用户关联。

一个用户可以拥有多条待办事项，待办事项属于一个用户，二者为一对多关系。

## 3. 数据表设计

### 3.1 用户表（users）

| 字段名 | 数据类型 | 是否为空 | 默认值 | 说明 |
|:---|:---|:---:|:---|:---|
| id | SERIAL | 否 | 自增 | 主键 |
| username | VARCHAR(50) | 否 | - | 用户名，唯一 |
| email | VARCHAR(100) | 否 | - | 邮箱，唯一 |
| password_hash | VARCHAR(255) | 否 | - | 密码哈希（bcrypt） |
| created_at | TIMESTAMPTZ | 否 | NOW() | 创建时间 |
| updated_at | TIMESTAMPTZ | 否 | NOW() | 更新时间 |

#### 约束

- 主键：`PRIMARY KEY (id)`
- 唯一约束：`UNIQUE (username)`、`UNIQUE (email)`
- 索引：`idx_users_email`、`idx_users_username`

#### Ent Schema 示例

```go
package schema

import (
    "time"

    "entgo.io/ent"
    "entgo.io/ent/schema/field"
    "entgo.io/ent/schema/index"
)

type User struct {
    ent.Schema
}

func (User) Fields() []ent.Field {
    return []ent.Field{
        field.Int("id").Positive().Unique(),
        field.String("username").NotEmpty().MaxLen(50),
        field.String("email").NotEmpty().MaxLen(100),
        field.String("password_hash").NotEmpty().MaxLen(255),
        field.Time("created_at").Default(time.Now),
        field.Time("updated_at").Default(time.Now).UpdateDefault(time.Now),
    }
}

func (User) Indexes() []ent.Index {
    return []ent.Index{
        index.Fields("username").Unique(),
        index.Fields("email").Unique(),
    }
}
```

### 3.2 待办事项表（todos）

| 字段名 | 数据类型 | 是否为空 | 默认值 | 说明 |
|:---|:---|:---:|:---|:---|
| id | SERIAL | 否 | 自增 | 主键 |
| title | VARCHAR(200) | 否 | - | 待办标题 |
| description | TEXT | 是 | NULL | 待办描述 |
| completed | BOOLEAN | 否 | false | 是否完成 |
| user_id | INTEGER | 否 | - | 外键，关联 users.id |
| due_date | TIMESTAMPTZ | 是 | NULL | 截止日期 |
| created_at | TIMESTAMPTZ | 否 | NOW() | 创建时间 |
| updated_at | TIMESTAMPTZ | 否 | NOW() | 更新时间 |

#### 约束

- 主键：`PRIMARY KEY (id)`
- 外键：`FOREIGN KEY (user_id) REFERENCES users(id) ON DELETE CASCADE`
- 索引：`idx_todos_user_id`、`idx_todos_completed`、`idx_todos_due_date`

#### Ent Schema 示例

```go
package schema

import (
    "time"

    "entgo.io/ent"
    "entgo.io/ent/schema/edge"
    "entgo.io/ent/schema/field"
    "entgo.io/ent/schema/index"
)

type Todo struct {
    ent.Schema
}

func (Todo) Fields() []ent.Field {
    return []ent.Field{
        field.Int("id").Positive().Unique(),
        field.String("title").NotEmpty().MaxLen(200),
        field.Text("description").Optional(),
        field.Bool("completed").Default(false),
        field.Int("user_id").Positive(),
        field.Time("due_date").Optional(),
        field.Time("created_at").Default(time.Now),
        field.Time("updated_at").Default(time.Now).UpdateDefault(time.Now),
    }
}

func (Todo) Edges() []ent.Edge {
    return []ent.Edge{
        edge.From("owner", User.Type).
            Ref("todos").
            Field("user_id").
            Unique().
            Required(),
    }
}

func (Todo) Indexes() []ent.Index {
    return []ent.Index{
        index.Fields("user_id"),
        index.Fields("completed"),
        index.Fields("due_date"),
    }
}
```

## 5. 数据库连接配置

### 5.1 环境变量

生成项目通过以下环境变量配置数据库连接：

| 环境变量 | 说明 | 示例值 |
|:---|:---|:---|
| DB_HOST | 数据库主机地址 | localhost |
| DB_PORT | 数据库端口 | 5432 |
| DB_USER | 数据库用户名 | postgres |
| DB_PASSWORD | 数据库密码 | secret |
| DB_NAME | 数据库名称 | todo_db |
| DB_SSL_MODE | SSL 模式 | disable |
| DB_MAX_OPEN_CONNS | 最大连接数 | 25 |
| DB_MAX_IDLE_CONNS | 最大空闲连接数 | 10 |

### 5.2 DSN 格式

PostgreSQL 连接字符串格式：

```text
postgres://<DB_USER>:<DB_PASSWORD>@<DB_HOST>:<DB_PORT>/<DB_NAME>?sslmode=<DB_SSL_MODE>
```

示例：

```text
postgres://postgres:secret@localhost:5432/todo_db?sslmode=disable
```

### 5.3 连接池配置

```go
import (
    "database/sql"
    "time"

    _ "github.com/lib/pq"
)

db, err := sql.Open("postgres", dsn)
if err != nil {
    return err
}

db.SetMaxOpenConns(25)
db.SetMaxIdleConns(10)
db.SetConnMaxLifetime(5 * time.Minute)
```

## 6. 数据库迁移

### 6.1 迁移策略

生成项目使用 Ent 的自动迁移功能（Schema Migration）。在开发阶段，可通过 `ent migrate` 命令或程序启动时自动执行迁移；在生产环境，建议使用版本化迁移文件（Versioned Migration）进行管理。

### 6.2 自动迁移代码

```go
package main

import (
    "context"
    "log"

    "github.com/example/todo/ent"
    _ "github.com/lib/pq"
)

func main() {
    client, err := ent.Open("postgres", dsn)
    if err != nil {
        log.Fatalf("failed opening connection to postgres: %v", err)
    }
    defer client.Close()

    ctx := context.Background()
    if err := client.Schema.Create(ctx); err != nil {
        log.Fatalf("failed creating schema resources: %v", err)
    }
}
```

### 6.3 Makefile 迁移命令

生成项目的 Makefile 提供以下迁移相关命令：

```makefile
migrate:
	go run ./cmd/server migrate

migrate-create:
	go run entgo.io/ent/cmd/ent migrate create $(name)
```

## 7. 索引设计

### 7.1 索引列表

| 表名 | 索引名 | 字段 | 类型 | 说明 |
|:---|:---|:---|:---:|:---|
| users | idx_users_username | username | 唯一索引 | 加速用户名查询，保证唯一性 |
| users | idx_users_email | email | 唯一索引 | 加速邮箱查询，保证唯一性 |
| todos | idx_todos_user_id | user_id | 普通索引 | 加速按用户查询待办 |
| todos | idx_todos_completed | completed | 普通索引 | 加速按状态过滤 |
| todos | idx_todos_due_date | due_date | 普通索引 | 加速按截止日期排序 |

### 7.2 索引设计说明

1. **users 表**：username 和 email 是高频查询字段，且需要保证唯一性，因此建立唯一索引。
2. **todos 表**：
   - `user_id` 是外键，查询用户名下所有待办时使用频繁；
   - `completed` 用于过滤已完成/未完成待办；
   - `due_date` 用于按截止日期排序和筛选。

## 8. 数据库安全

### 8.1 密码存储

用户密码必须经 bcrypt 算法哈希后存储，禁止明文保存。示例：

```go
import "golang.org/x/crypto/bcrypt"

hash, err := bcrypt.GenerateFromPassword([]byte(password), bcrypt.DefaultCost)
```

### 8.2 连接安全

1. 生产环境应启用 SSL/TLS 连接（`sslmode=require`）。
2. 数据库密码通过环境变量注入，禁止硬编码在源代码中。
3. 使用连接池限制最大连接数，防止资源耗尽。

### 8.3 数据备份

1. 开发阶段：通过 `pg_dump` 定期备份数据库。
2. 生产阶段：建议使用数据库托管服务提供的自动备份功能。

## 9. 多数据库支持

### 9.1 MySQL 支持

当用户选择 MySQL 时，生成项目使用 `github.com/go-sql-driver/mysql` 驱动，DSN 格式为：

```text
<DB_USER>:<DB_PASSWORD>@tcp(<DB_HOST>:<DB_PORT>)/<DB_NAME>?parseTime=true
```

### 9.2 SQLite 支持

当用户选择 SQLite 时，生成项目使用 `github.com/mattn/go-sqlite3` 驱动，数据库文件位于项目目录下：

```text
./data/app.db
```

SQLite 主要用于开发和测试环境，不推荐用于生产环境。

## 11. 附录

### 11.1 数据库初始化 SQL

```sql
CREATE DATABASE todo_db;

\c todo_db;

CREATE TABLE users (
    id SERIAL PRIMARY KEY,
    username VARCHAR(50) NOT NULL UNIQUE,
    email VARCHAR(100) NOT NULL UNIQUE,
    password_hash VARCHAR(255) NOT NULL,
    created_at TIMESTAMPTZ NOT NULL DEFAULT NOW(),
    updated_at TIMESTAMPTZ NOT NULL DEFAULT NOW()
);

CREATE TABLE todos (
    id SERIAL PRIMARY KEY,
    title VARCHAR(200) NOT NULL,
    description TEXT,
    completed BOOLEAN NOT NULL DEFAULT FALSE,
    user_id INTEGER NOT NULL REFERENCES users(id) ON DELETE CASCADE,
    due_date TIMESTAMPTZ,
    created_at TIMESTAMPTZ NOT NULL DEFAULT NOW(),
    updated_at TIMESTAMPTZ NOT NULL DEFAULT NOW()
);

CREATE INDEX idx_todos_user_id ON todos(user_id);
CREATE INDEX idx_todos_completed ON todos(completed);
CREATE INDEX idx_todos_due_date ON todos(due_date);
```
