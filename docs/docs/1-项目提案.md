# 项目提案/商业计划书

## 1. 项目概述

### 1.1 项目名称

**Golang AI 原生全栈应用快速开发脚手架**（以下简称"Go Scaffold"）。

### 1.2 项目背景

在当今云原生与 AI 辅助编程快速发展的背景下，Go 语言已成为构建高性能后端服务的首选语言之一。根据行业统计，Go 在新增云原生 Web 服务中占据约 42% 的份额，而以 Gin、Echo、Fiber 为代表的 Web 框架生态日趋成熟。与此同时，AI 能力正加速渗透到开发工具链中：Google Genkit Go、CloudWeGo Eino 等框架将大语言模型（LLM）编排能力引入 Go 生态，使得"自然语言生成代码"从概念逐步落地为可用工具。

然而，对于高校学生、独立开发者以及中小型团队而言，启动一个新的 Go 全栈项目仍然存在较高的门槛：需要手动配置 Web 框架、ORM、数据库、认证、日志、OpenAPI、Docker、CI/CD 等多个环节，重复劳动多、配置复杂、学习曲线陡峭。市场上虽然已有一些脚手架工具（如 Cookiecutter、Yeoman、GoFrame CLI），但大多停留在模板拷贝层面，缺乏与本地 LLM 的深度融合，也难以根据自然语言描述动态生成业务代码。

本项目旨在构建一个面向 Go 生态的 AI 原生全栈应用快速开发脚手架，通过 CLI 工具将项目初始化、模板渲染、AI 代码生成、依赖管理、部署配置等环节自动化，帮助后端开发者在 5 分钟内启动一个可运行的全栈项目，显著降低重复劳动，提升开发效率。

### 1.3 项目目标

本项目的主要目标包括：

1. **构建交互式 CLI 工具**：基于 Cobra 和 Bubble Tea 提供友好的命令行交互体验，支持 `init`、`generate`、`build` 三大核心命令。
2. **提供可扩展的项目模板**：支持 Echo/Fiber/Gin 后端框架、Ent/sqlc/GORM ORM、PostgreSQL/MySQL/SQLite 数据库、React/Vue/Svelte 前端模板的组合选择。
3. **集成 AI 代码生成能力**：基于 Genkit Go 框架对接本地 Ollama 模型（如 ornith:9b），实现根据自然语言描述生成 CRUD 代码、handler、数据库模型和 JWT 认证逻辑。
4. **自动生成 OpenAPI 3.1 文档**：通过 Huma 框架或模板方式，确保生成的后端项目具备规范的 API 文档。
5. **提供开箱即用的部署配置**：自动生成 Dockerfile、docker-compose.yml、GitHub Actions CI/CD 工作流，支持本地开发、容器化部署和云发布。
6. **验证真实可用性**：使用本脚手架生成一个完整的 Demo 应用（如待办事项管理系统），并验证其可编译、可运行、可部署。

### 1.4 预期成果

本项目的预期成果包括：

- 一个可独立运行的 Go CLI 工具 `go-scaffold`，支持 Windows、macOS、Linux 三个平台。
- 一套可扩展的项目模板仓库，覆盖后端、前端、部署配置等常见场景。
- 一份完整的设计报告和用户使用文档。
- 一个基于本脚手架生成的 Demo 应用及其运行截图。
- 项目源代码仓库（GitHub）和预编译二进制发布包。

## 2. 市场分析

### 2.1 目标用户

本项目的目标用户主要包括：

1. **高校计算机相关专业学生**：在学习 Go Web 开发、软件工程实训等课程时，需要快速搭建项目原型，降低环境配置成本。
2. **独立开发者与自由职业者**：希望快速启动个人项目或外包项目，减少重复的基础代码编写工作。
3. **中小型技术团队**：在项目早期需要快速验证业务想法，统一团队项目结构和开发规范。
4. **后端开发者**：熟悉 Go 但不熟悉前端或 DevOps，希望通过脚手架一键生成全栈项目骨架。

### 2.2 市场需求

当前市场上常见的脚手架工具可分为以下几类：

| 工具类型 | 代表产品 | 优势 | 不足 |
|:---:|:---|:---|:---|
| 通用模板生成器 | Cookiecutter、Yeoman | 模板丰富、跨语言 | 对 Go 生态支持有限，缺乏 AI 能力 |
| Go 框架自带 CLI | GoFrame CLI、Beego CLI | 深度集成框架 | 绑定特定框架，不够灵活 |
| AI 代码编辑器插件 | GitHub Copilot、Codeium | 代码补全能力强 | 需要付费或联网，不生成项目骨架 |
| 全栈启动模板 | create-t3-app、RedwoodJS | 前端体验好 | 主要针对 Node.js 生态 |

从表中可以看出，面向 Go 生态、融合本地 LLM、支持自然语言生成代码的全栈脚手架工具仍存在明显空白。本项目正是针对这一空白，提供一个开源、本地化、可扩展的解决方案。

### 2.3 竞争分析

本项目的核心竞争优势在于：

1. **AI 原生设计**：不同于传统脚手架的静态模板拷贝，本项目通过 Genkit Go 对接本地 Ollama，能够根据用户输入动态生成业务代码。
2. **本地化运行**：所有 AI 推理均在本地完成（Ollama + 量化模型），无需联网、无需 API Key，保护代码隐私。
3. **Go 生态深度集成**：优先支持 Echo + Ent + PostgreSQL 组合，兼顾性能、类型安全和可维护性。
4. **可扩展架构**：模板引擎、AI 流程、命令解析器分层设计，便于后续支持更多框架和模型。

## 3. 预期成本与收益

### 3.1 预期成本

本项目为课程实训项目，主要成本为人力成本和开发设备成本，不涉及大规模商业采购。

| 成本项 | 说明 | 估算 |
|:---|:---|---:|
| 开发人力 | 5 人小组，2 周（10 个工作日） | 主要成本 |
| 开发设备 | 个人笔记本电脑 | 已有 |
| 本地模型运行 | Ollama + ornith:9b 量化版 | 免费开源 |
| 本地部署环境（可选） | Docker 本地部署验证 | 0 元（复用本地开发机） |
| 代码托管 | GitHub 公开仓库 | 免费 |

### 3.2 预期收益

1. **教学收益**：完成软件工程实训考核，积累 Go 语言、CLI 工具开发、AI 应用集成等实践经验。
2. **工具收益**：产出一个可直接复用的开源工具，后续可用于个人项目或团队项目初始化。
3. **技能收益**：掌握 Cobra、Bubble Tea、Genkit Go、Ent、Echo、Docker 等主流技术栈。
4. **社区收益**：若项目质量达到预期，可发布到 GitHub 并吸引社区贡献，形成长期价值。

## 4. 风险分析

| 风险项 | 可能性 | 影响 | 应对措施 |
|:---|:---:|:---:|:---|
| AI 生成代码质量不稳定 | 中 | 高 | 设计严格的 Prompt 模板，使用 go/ast 校验生成代码语法，失败时重试或给出提示 |
| 本地模型运行资源不足 | 低 | 中 | 默认使用 ornith:9b（5.6GB），资源不足时降级到 qwen3.5:4b（~2.5GB） |
| 多框架模板维护复杂 | 中 | 中 | 2 周内优先实现 Echo + Ent + React 模板，其他作为扩展参考 |
| 第三方依赖版本更新 | 低 | 低 | 固定主要依赖版本，在文档中说明兼容性 |

## 5. 结论

本项目紧密结合当前 Go 语言云原生发展和 AI 辅助编程趋势，具有较高的技术前瞻性和实用价值。通过构建一个 AI 原生的全栈应用快速开发脚手架，不仅能够显著提升开发效率，还能让学生在有限的项目周期内接触到 CLI 工具设计、模板引擎、AI 编排、DevOps 配置等多个工程实践领域。项目目标明确、技术路线清晰、风险可控，建议启动实施。
